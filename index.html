Some agreements

  • Components' names should be Capitalized.
  • Tag-names should be with lowercase.
  • React DOM uses camelCase for property naming.
  • JSX is like XML. Every tag should be closed.
  • Component's inner content can be taken via children property.

Simple functional component

		function MyComponent({ name = 'World' }) {
		  return <div>Hello, {name}.</div>;
		}

Functional component's pros

  • Pure
  • No boilerplate
  • Can be reused as plain function
  • Stateless
  • Very simple to read

Functional component's cons

  • Stateless
  • Slow in some cases (use recompose.pure)
  • No lifecycle hooks

Stateless component via classes

			class MyComponent extends React.Component {
			  static defaultProps = { name: 'World' }
			  render() {
			    const { name } = this.props;
			    return <div>Hello, {name}.</div>;
			  }
			}

Class's pros

  • Lifecycle hooks are allowed
  • It's ES2015 classes
  • Any methods you want
  • Stateless or stateful

Class's cons

  • Boilerplate code
  • It takes much memory that functional component

Control the component

  • Props - passed from parent component
  • State - internal info about this component
  • Context - passed from any parent component

Context is not very recommended for use. So I will not describe it.

Access component's state

		class MyComponent extends React.Component {
		  state = { updatedAt: new Date(), createdAt: new Date() }
		  render() {
		    const { updatedAt } = this.state;
		    return <div>
		      {createdAt.toString()}-{updatedAt.toString()}
		    </div>;
		  }
		}

Event handling

		  handleClick() { console.log(this); }

		      Click me

		  }
		}

Event handling

		class MyComponent extends React.Component {
		  constructor(props) {
		    super(props);
		    this.handleClick = this.handleClick.bind(this);
		  }
		  handleClick => () => { console.log(this); } 
		  render() {
		    return <button onClick={() => this.handleClick()} />;
		  }
		}

Event handling
